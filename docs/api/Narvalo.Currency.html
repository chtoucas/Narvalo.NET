Struct Currency

Represents a currency such as Euro or US Dollar.

Syntax
public struct Currency : ICurrencyUnit, IEquatable<Currency>
Remarks

Recognized currencies are defined in ISO 4217.

Only decimal currencies are supported.

You can not directly construct a currency. You must instead use one of the static factories, eg Of(String).

This class does not offer extended information about the currency.

Remarks

If the currency has no minor units (null), which is the case for meta-currencies (but not for all of them), we return 0.

If we don't known whether the currency has a minor currency unit or not, which is the case for all legacy currencies, we use 28 (the maximum scale for a decimal) as a replacement, ie an amount in this currency is free to take any value in the decimal range.

HasMinorCurrency

Gets a value indicating whether the currency admits a minor currency unit.

Declaration
public bool HasMinorCurrency { get; }
Property Value
Type Description
System.Boolean
Remarks

We consider that legacy currencies do not admit a minor unit. This is actually false, but we do not have enough informations at our disposal to give a proper answer. It is not only because ISO 4217 does not give us the data, but also because a currency could have changed over time (devaluation, decimalisation...) while keeping the same main unit (which most certainly didn't even exist at the time).

IsMetaCurrency

Gets a value indicating whether the currency is a meta-currency.

Declaration
public bool IsMetaCurrency { get; }
Property Value
Type Description
System.Boolean

true if the currency is a meta-currency; otherwise, false.
Remarks

Meta-currencies designates most supranational currencies: regional currencies that are not attached to a specific country (nevertheless notice that EUR is not part of them, and there is more if you add withdrawn currencies), precious metals, the test currency, the "no" currency and currencies used in international finance.

Meta-currencies are not attached to a specific country. Their numeric codes are in the range 900-999 and their codes are in the range XA(A)-XZ(Z). They fall in the ranges of user-assigned codes as defined by the ISO 3166 standard, ie they will never clash with those of a real country.

RegisterCurrency(String, Nullable<Int16>)

Register a currency not part of ISO 4217.

It can also be useful when the library is not up-to-date with the ISO 4217 list of active currencies.

Declaration
public static bool RegisterCurrency(string code, short ? minorUnits)
Parameters
Type Name Description
System.String code

The three letters code.
System.Nullable<System.Int16> minorUnits

The number of minor units; null if the currency does not have a minor currency unit and UnknownMinorUnits if the status is unknown.
Returns
Type Description
System.Boolean

true if the operation succeeded; otherwise, false.
Remarks

All currencies registered via this method have the UserDefined type. In particular, they inherit the Circulating type; as a consequence, you can not register a withdrawn currency.

If you have more than one currency to register, you should use RegisterCurrencies(Dictionary<String, Nullable<Int16>>) instead.

This method is thread-safe.

Exceptions
Type Condition
System.ArgumentException

Thrown if the candidate currency does not meet the requirements: a code must be of length 3 and made of ASCII uppercase letters, and minorUnits, if not null, must be greater than or equal to zero and lower than or equal to 28.
